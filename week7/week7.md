## Условие

Процесс-родитель создает процесса-сына, а тот в свою очередь процесса-внука. Процесс-родитель и процесс-внук должны быть соединены анонимным каналом в направлении от родителя к внуку.

Процесс-родитель считывает 32-битные знаковые целые числа, подаваемые на стандартном потоке ввода в текстовом виде. Последовательность заканчивается признаком конца файла. Процесс-родитель передает считанные числа в канал в бинарном виде.

Процесс-внук считывает числа из канала и вычисляет их сумму. После чтения всех чисел процесс-внук выводит на стандартный поток вывода их сумму и завершает работу.

Процесс-родитель должен дождаться завершения всех порожденных им процессов и завершиться сам с кодом завершения 0.

### Пример 1

#### Входные данные


#### Результат работы

## Условие

Программе в аргументах командной строки передается произвольное (возможно пустое) количество команд для выполнения. Программа должна запустить на выполнение процессы, чтобы они выполнялись в конфигурации, соответствующей команде shell:

```sh
cmd1 | cmd2 | ... | cmdN
```

Родительский процесс должен дождаться завершения всех созданных им процессов и сам завершить выполнение. Если все системные вызовы, исполнявшиеся родителем (например, создание процесса), завершились успешно, родитель завершается с кодом 0. В случае ошибки выполнения системного вызова (например, невозможности создания очередного процесса), родитель уничтожает уже созданные к этому моменту процессы с помощью сигнала SIGKILL и сам завершается с кодом 1 после завершения убитых им сыновей.

Команды cmd1, cmd2, ... запускайте на выполнение с помощью execlp.

В родителе одновременно может быть открыто не более 6 файловых дескрипторов.

Уничтожение процесса выполняйте с помощью системного вызова kill:

```c
kill(pid, SIGKILL);
```

Например, если скомпилированная программа называется solution, то ее запуск с аргументами

```sh
./solution ls cat wc
```

должен дать такой же результат на стандартном потоке вывода, как выполнение команды shell

```sh
ls | cat | wc
```

## Условие

Родитель создает двух сыновей, связывая их двумя каналами. Сыновья начинают обмениваться числами 1, 2, 3, ... Число 1 получает первый сын. При получении числа из входного канала каждый сын печатает на стандартный поток вывода родителя свой номер (1 или 2) и полученное число и пересылает обратно число, на 1 большее. При достижении числа, задаваемого в командной строке, сын не пересылает число в канал, а завершает работу.

Процесс может завершить работу либо при получении максимального числа из канала, либо при наступлении конца файла в канале. Завершать процесс в других случаях не разрешается.

Отец дожидается завершения работы обоих процессов, затем выводит строку `Done` и завершает работу.

Например, если в командной строке передано число 5, то вывод должен быть таким:

```text
1 1
2 2
1 3
2 4
Done
```

## Дополнительные требования

Данные передавайте в текстовом виде, для ввода-вывода используйте высокоуровневый ввод-вывод (`printf`, `scanf` и т. д.).
Привязывайте файловые дескрипторы к дескрипторам потока с помощью функции `fdopen`.

## Условие

В аргументах командной строки задается число N процессов (N > 0), и число жизней у каждого процесса L (L > 0).

Главный процесс должен создать N процессов, пронумерованных от 0 до N - 1. Далее главный процесс считывает со стандартного потока ввода последовательность 32-битных беззнаковых чисел. Номер процесса определяется как остаток от деления считанного числа на N. Главный процесс "стреляет" в этот процесс. Соответствующий процесс выводит на стандартный поток вывода свой номер и количество оставшихся жизней. Если процесс "убит", то есть у него не осталось жизней, процесс завершает работу. Выстрел в убитый процесс не выполняет никаких действий.

После окончания ввода со стандартного потока ввода отец выводит число -1 и число оставшихся в живых процессов, после чего все оставшиеся процессы должны завершиться добровольно, после чего завершается отец.

Например, если программа запущена с аргументами:

```sh
./solution 2 3
```

и на стандартный поток ввода подается

```text
0 1 1 1
```

то на стандартный поток вывода должно быть напечатано

```text
0 2
1 2
1 1
1 0
-1 1
```

Вывод процессов на стандартном потоке вывода должен быть строго в порядке выстрела в них, то есть если сначала был выстрел в процесс 1, а после него был выстрел в процесс 2, то на стандартном потоке вывода сначала должна быть информация от процесса 1, а затем информация от процесса 2.

## Дополнительные требования

Для взаимодействия процессов используйте только неименованные каналы, включая момент обработки конца ввода.
Процесс-родитель не должен хранить информацию о произведённых выстрелах, в частности счётчик жизней.
Если счётчик жизней процесса-сына становится равным 0, он должен завершаться немедленно.
Процесс-сын должен выводить число оставшихся жизней немедленно после выстрела.

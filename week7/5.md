## Условие

В аргументах командной строки задается число N процессов (N > 0), и число жизней у каждого процесса L (L > 0).

Главный процесс должен создать N процессов, пронумерованных от 0 до N - 1. Далее главный процесс считывает со стандартного потока ввода последовательность 32-битных беззнаковых чисел. Номер процесса определяется как остаток от деления считанного числа на N. Главный процесс "стреляет" в этот процесс. Соответствующий процесс выводит на стандартный поток вывода свой номер и количество оставшихся жизней. Если процесс "убит", то есть у него не осталось жизней, процесс завершает работу. Выстрел в убитый процесс не выполняет никаких действий.

После окончания ввода со стандартного потока ввода отец выводит число -1 и число оставшихся в живых процессов, после чего все оставшиеся процессы должны завершиться добровольно, после чего завершается отец.

Например, если программа запущена с аргументами:

```sh
./solution 2 3
```

и на стандартный поток ввода подается

```text
0 1 1 1
```

то на стандартный поток вывода должно быть напечатано

```text
0 2
1 2
1 1
1 0
-1 1
```

Вывод процессов на стандартном потоке вывода должен быть строго в порядке выстрела в них, то есть если сначала был выстрел в процесс 1, а после него был выстрел в процесс 2, то на стандартном потоке вывода сначала должна быть информация от процесса 1, а затем информация от процесса 2.

## Дополнительные требования

Для взаимодействия процессов используйте только неименованные каналы, включая момент обработки конца ввода.
Процесс-родитель не должен хранить информацию о произведённых выстрелах, в частности счётчик жизней.
Если счётчик жизней процесса-сына становится равным 0, он должен завершаться немедленно.
Процесс-сын должен выводить число оставшихся жизней немедленно после выстрела.

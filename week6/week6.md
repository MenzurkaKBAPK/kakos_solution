Первая буква 'p' в аргументе командной строки означает, что эта программа должна работать параллельно с другими программами, помеченными 'p'. Первая буква 's' обозначает, что эта программа работает последовательно, то есть запускается на выполнение после того, как все программы в предыдущих аргументах командной строки завершены. Главная программа должна дождаться завершения выполнения всех процессов и вывести на стандартный поток вывода число программ, отработавших успешно. Процесс считается отработавшим успешно, если он завершился с кодом завершения 0. Программы запускать на выполнение с помощью execlp.

Предполагать, что все системные вызовы завершаются успешно, нельзя.

## Условие

Напишите функцию:

```c
int mysystem(const char *cmd);
```

В параметре cmd передается командная строка для запуска. Необходимо обработать командную строку, запустить заданную программу на выполнение и дождаться её завершения. Разбор командной строки должен быть реализован полностью в программе без привлечения внешних программ, таких как /bin/sh.

Разбор командной строки выполняется следующим образом:
Строка просматривается посимвольно слева направо. При обнаружении байта с кодом 0x27 (апостроф) байты до следующего байта с кодом 0x27 остаются без изменений. Иначе если обнаружен последовательность байт 0x24 (знак доллара), 0x7b (открывающая фигурная скобка), следующая за ними последовательность байт 0x7d (закрывающая фигурная скобка) рассматривается как имя переменной окружения и заменяется на её значение. Если переменная окружения не определена, она заменяется на пустую строку. При подстановке переменных окружения байты 0x24, 0x7b и 0x7d и имя переменной окружения из строки удаляются.

Таким образом, подстановки переменных окружения не выполняются внутри фрагментов в апострофах, например `str' i' 'st'${bb}` в исходных частях строки выполняется только один проход по подстановкой переменных окружения. Например, строка `1'${USER}'2${USER}4'${USER}'` преобразовывается в строку `1'${USER}'2здравствуйте'4'${USER}'`, если переменная окружения USER равна строке `здравствуйте`, а в противном случае в строке останется `1'${USER}'2'${USER}'4'${USER}'`. Этот пример нужен лишь для демонстрации правила; строка в фигурные скобки допустима и для пустых строк.

Строка разбивается на токены. Токены разделяются произвольным количеством пробельных символов. Пробельные символы также могут находиться в начале и конце строки. Но если при просмотре строки слева направо обнаруживается байт 0x27 (апостроф) или 0x22 (кавычка) байты между этой и следующей такой же без изменений до парной байта 0x27 или 0x22. Эти парные байты удаляются из токена. Внутри фрагмента в апострофах могут быть кавычки, а внутри фрагмента в кавычках - апострофы. Эта информация в токене не сохраняется.

Например, строка `a b'" c "d e 'f' g"` преобразуется в последовательность токенов a, b c, d e f. Токен может быть пустым, если, например, он записан с помощью пары апострофов, значение пустого токена — строка длины 0.

Токены просматриваются слева направо. Если в токене есть байт 0x3d (знак равенства), все байты слева от первого в токене байта 0x3d считаются именем переменной окружения, а все байты справа от первого в токене байта 0x3d — значением переменной окружения. Такие токены задают изменения в окружении для запускаемой программы относительно текущего окружения. Изменения в окружении, образованные соседними токенами, если несколько токенов подряд модифицируют одну и ту же переменную окружения, эффект имеет самая последняя модификация. Переменные окружения, заданные таким образом, действуют только на запускаемую программу и не влияют на окружение текущего процесса.

Первым по порядку токеном в котором нет байта 0x3d является именем программы для запуска. Для определения полного пути к запускаемому файлу используется переменная PATH, возможно модифицированная предыдущими токенами. Для поиска программы в путях, задаваемых PATH, может использоваться соответствующие функции семейства exec. Следующие за именем программы токены являются аргументами командной строки запускаемой программы.

Например, в последовательности токенов A=2, BBB=10, ls, -l, /proc первые два токена задают переменные окружения A и BBB для запускаемой программы ls. Полное имя программы ls это имя исполняемого файла, /proc - аргументы командной строки.

Если командная строка некорректна, функция должна вернуть код -1. Если создать процесс не удаётся, функция должна вернуть -1. Если указанную команду запустить не удалось, функция должна вернуть код 127. Если созданный процесс завершился из-за сигнала, необходимо вернуть номер этого сигнала, к которому прибавлено 1024. В противном случае необходимо вернуть код завершения дочернего процесса. В любом случае, только значение 0 может обозначать нормальное выполнение созданного процесса.

# week09-5-signal-5

## Условие задачи

В аргументах командной строки передаётся имя программы для запуска `PROG` и целое число K (0 < K <= 100000).

Запустите на выполнение программу `PROG`, связав её с текущей программой с помощью двух именованных каналов (fifo). Один канал перенаправлен на стандартный поток ввода программы `PROG`, другой канал перенаправлен на стандартный поток вывода программы `PROG`.

Две программы обмениваются 32-битными беззнаковыми целыми числами в текстовом виде. Когда ваша программа получает от `PROG` некоторое число X, она выводит его на стандартный поток вывода на отдельной строке, а затем отвечает программе `PROG` числом X+1. Если запись в канал, передающий данные программе `PROG` неуспешна, дальнейшее чтение данных от `PROG` не выполняется. Программа `PROG` может завершиться сама в любой момент.

Если за последнюю астрономическую секунду от программы `PROG` было получено более чем K чисел, первое же полученное число, превышающее лимит, не выводится, но ваша программа выводит на стандартный поток вывода число 1000000, и взаимодействие с `PROG` прекращается: ваша программа должна закрыть записывающую сторону канала и дожидаться завершения `PROG`. Например, при K==1 между получениями последовательных чисел от `PROG` должно пройти не менее секунды астрономического времени, а при K==2 при получении $i$-го числа должно пройти не менее секунды астрономического времени от получения $i-2$ числа. Таким образом, при K==2 если программа `PROG` вывела на свой стандартный поток вывода числа 1, 2, 3 быстрее, чем за одну секунду, ваша программа должна вывести на стандартный поток вывода

```
1
2
1000000
```

Ваша программа должна дождаться завершения работы программы `PROG` следующим образом. Если после закрытия с вашей стороны записывающей стороны канала программа `PROG` не успела завершиться за 200 мс (миллисекунд), ваша программа должна вывести на стандартный поток вывода число 1000001 и отправить программе `PROG` сигнал `SIGTERM`.

Если программа `PROG` не успела завершиться за 200 мс от момента отправки `SIGTERM`, ваша программа должна вывести на стандартный поток вывода число 1000002 и отправить сигнал `SIGKILL`.

Ожидание завершения работы программы `PROG` должно прекращаться сразу же после её завершения. Ждать всегда 200 мс недопустимо.

В любом случае ваша программа должна завершиться после завершения `PROG` и всегда с кодом 0.

Ваша программа может не иметь прав записи в текущий каталог. Каналы должны быть защищены от вмешательства других пользователей.

Для отслеживания времени получения чисел используйте кольцевой буфер. Для ожидания завершения программы `PROG` используйте интервальный таймер реального времени, устанавливаемый с помощью `setitimer`.

## Приверы вывода

### Пример 1

Программа `PROG` вывела числа 1, 2, 3, не превысила лимита и завершилась сама.

```
1
2
3
```

### Пример 2

Программа `PROG` вывела числа 1, 2, 3, но превысила лимит на числе 4.

```
1
2
3
1000000
```

### Пример 3

Программа `PROG` вывела числа 1, 2, 3, но превысила лимит на числе 4, затем не успела завершиться за 200 мс после закрытия канала.

```
1
2
3
1000000
1000001
```

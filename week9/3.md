# week09-3-signal-3

## Условие

Родитель создает двух сыновей, связывая их неименованным каналом. Сыновья начинают обмениваться числами 1, 2, 3, ... Число 1 получает первый сын. При получении числа из входного канала каждый сын печатает на стандартный поток вывода родителя свой номер (1 или 2) и полученное число и пересылает обратно число, на 1 большее. При достижении числа, задаваемого в командной строке, пересылает то же самое число в канал и завершает работу.

Процесс должен завершить работу при получении максимального числа из канала. Завершать процесс в других случаях не разрешается.

Отец дожидается завершения работы обоих процессов, затем выводит строку `Done` и завершает работу.

Если в командной строке передано число 5, то вывод должен быть таким:

```
1 1
2 2
1 3
2 4
Done
```

## НФТ

- Сыну запрещается отправлять сигнал отцу.
- Данные по каналу передавать в текстовом виде.
- Для ввода-вывода в канал использовать высокоуровневый ввод-вывод (`printf`, `scanf`).
- Для синхронизации использовать единственный сигнал `SIGUSR1`.
- PID процессов передавать через канал.
- Для работы с сигналами использовать только `signal` или `sigaction` и `pause` или `usleep`.

*Language: gcc-GNU C 15.2.0*

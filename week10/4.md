# week10-4-signal-10

## Условие

В аргументе командной строки задаётся число N — число процессов (N < 100). Процесс-родитель создаёт N процессов, пронумерованных от 0 до N-1.

На стандартном потоке ввода подаётся последовательность беззнаковых целых 32-битных чисел. Процессы работают по одному, первым работает процесс 0. Очередной работающий процесс считывает очередное число со стандартного потока ввода, выводит номер процесса (себя) и считанное число, вычисляет номер следующего процесса как остаток от деления считанного числа на N и передаёт очерёдность хода следующему процессу. При достижении конца файла все созданные процессы должны завершиться.

Для передачи хода следующему процессу используйте сигнал `SIGUSR1`. Для работы с сигналами используйте `signalfd`. Pid процессов храните в разделяемой памяти.

## Пример

При N == 6 если на стандартном потоке ввода подаются числа

```
1
4
9
16
25
36
49
64
81
100
121
144
```

На стандартном потоке вывода должно быть напечатано

```
0 1
1 4
4 9
3 16
4 25
1 36
0 49
1 64
4 81
3 100
4 121
1 144
```

# week10-5-signal-11

## Условие

Реализуйте поддержку массива неограниченного размера, а точнее массива, расширяющегося неявно при добавлении элементов в его конец.

Напишите функции

```
void *create_array();
void free_array(void *ptr);
```

Функция create_array создаёт новый пустой массив и возвращает адрес его начала. При ошибке возвращается `NULL`. Функция `free_array` освобождает всю память, занимаемую массивом, возвращая её операционной системе. Если `ptr == NULL` функция не делает ничего.

Работать с массивом неограниченного размера можно следующим образом:

```
    int *p = create_array();
    for (int i = 0; i < n; ++i) {
        p[i] = i;
    }
```

здесь `n` может быть любым, пока хватает памяти.

Под элементы пустого массива память не выделяется. При обращении к элементу с индексом 0 пустого массива сразу выделяется одна страница виртуальной памяти, и в дальнейшем память, выделяемая под массив, расширяется по одной странице при необходимости. Расширение массива гарантированно происходит только при добавлении элемента в его текущий конец. Если индекс обращения к элементу массива находится далеко за его концом, за границей ещё не отображённой страницы, непосредственно идущей после отображенных страниц массива, программа должна получить `SIGSEGV`.

Перемещение массива в памяти не поддерживается. Если невозможно выделить очередную страницу для расширения массива, программа должна получить сигнал `SIGBUS`. Но стратегия начального размещения массива должна стремиться минимизировать такие ситуации.

Должны поддерживаться до 32 неограниченных массива одновременно. При попытке создания 33-го массива функция создания возвращает `NULL`. Функция `free_array` освобождает слот для массива.

Функция `create_array` должна возвращать адрес, выровненный по размеру страницы. Для проверки возможности расширения массива можно использовать `MAP_FIXED_NOREPLACE`.

*Language: gcc-GNU C 15.2.0*
